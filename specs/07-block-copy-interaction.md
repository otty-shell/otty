# 07. Минимальное взаимодействие: копирование активного блока

## Обзор

Седьмая задача: добавить простое действие над активным блоком — копирование содержимого выбранного блока в буфер обмена. Используется `selected_block_id` из `TerminalViewState` и данные snapshot’а.

См. спецификацию: `specs/spec.md`, раздел 8.3.

## Что уже есть

- Задачи 01–06:
  - UI подсвечивает рамки только для блоков `Command`/`FullScreen`, а подсветка промптов отключена.
  - `TerminalViewState` уже хранит `hovered_block_id`, `hovered_block_kind`, `selected_block_id` и `selected_block_kind`.
  - При начале обычного выделения текста блоковый выбор автоматически сбрасывается.
- `SnapshotView`:
  - предоставляет `blocks()`/`block_at_point()`, а `BlockSnapshot.start_line` и `SnapshotCell.point.line` находятся в одной системе координат (учитывая `display_offset`).
- `InputManager`:
  - в `handle_cursor_moved` уже обновляет `hovered_block_*` через `block_at_point`.
  - игнорирует клики по `BlockKind::Prompt`, поэтому пользователь может выбрать только `Command` или `FullScreen`.
  - реализует `BindingAction::Copy` для копирования выделенного текста.
  - имеет доступ к `SnapshotOwned` через `terminal_state` и умеет работать с clipboard.

## Что нужно сделать

1. Кнопка действий блока:
   - В `TerminalView` добавить визуальную кнопку (иконку «⋮») в правом верхнем углу блока.
   - Кнопка отображается только если блок hovered или selected, и скрывается для `BlockKind::Prompt`.
   - При наведении/клике подсвечивается (изменение цвета/альфы).
   - Геометрию кнопки нужно синхронизировать между отрисовкой и обработкой мыши (лучше вынести расчёт в helper).

2. Обработка мыши в `InputManager`:
   - В `handle_cursor_moved` вычислять hitbox кнопки:
     - ориентироваться на `hovered_block_id` либо `selected_block_id`;
     - если курсор внутри, выставлять `view_state.hovered_action_block_id` и запрашивать перерисовку.
   - В `handle_mouse_event` / `handle_left_button_pressed`:
     - если клик пришёл по кнопке, не запускать текстовое выделение/MouseReport;
     - вместо этого собрать текст блока и записать в clipboard (см. пункт 3).
     - после копирования можно оставить блок выделенным.

3. Сбор текста блока:
   - Реализовать helper (можно приватный метод `InputManager`), который:
     - принимает `SnapshotView` и `block_id`;
     - находит `BlockSnapshot`, убеждается, что `line_count > 0` и `kind != Prompt`;
     - итерирует `snapshot.cells`, выбирая ячейки с `point.line` в диапазоне блока;
     - пропускает `Flags::WIDE_CHAR_SPACER`, чтобы широкие символы не дублировались;
     - группирует символы по строкам, обрезая хвостовые пробелы и добавляя `\n` между строками.
   - Helper возвращает `Some(String)` при успехе; вызывающий код пишет результат в clipboard (`ClipboardKind::Standard`) и возвращает `iced::event::Status::Captured`.

4. Поведение для различных типов блоков:
   - `Command` и `FullScreen` копируются как описано выше.
   - `Prompt` по-прежнему игнорируются: кнопка не показывается, helper возвращает `None`.

## Ожидаемый результат

- Пользователь может:
  - навести курсор на блок или выбрать его кликом;
  - увидеть кнопку с тремя точками в правом верхнем углу;
  - нажать её и получить в буфере обмена текст именно этого блока (без соседних блоков); промпт-блоки игнорируются.
- Копирование работает для текущего viewport — если часть блока скрыта скроллом, копируется только видимая часть (ограничение итерации 1).

## Как протестировать

1. Юнит‑тесты:
   - В тестах `InputManager` создать `SnapshotOwned` с несколькими блоками, заполненными текстом (включая широкие символы и строки разной длины).
   - Смоделировать выбор/hover блока, вызвать helper копирования и проверить содержимое тестового clipboard.
   - Дополнительно: блок вне viewport (`line_count = 0`) и `Prompt` → helper возвращает `None`.

2. Ручной тест:
   - Запустить терминал, выполнить несколько команд (создать несколько блоков);
   - навести курсор на блок или выбрать его кликом;
   - нажать кнопку «⋮»;
   - вставить содержимое в текстовый редактор и убедиться, что скопировался только выбранный блок.
