# 07. Минимальное взаимодействие: копирование активного блока

## Обзор

Седьмая задача: добавить простое действие над активным блоком — копирование содержимого выбранного блока в буфер обмена. Используется `selected_block_id` из `TerminalViewState` и данные snapshot’а.

См. спецификацию: `specs/spec.md`, раздел 8.3.

## Что уже есть

- Задачи 01–06:
  - UI подсвечивает рамки только для блоков `Command`/`FullScreen`, а подсветка промптов отключена.
  - `TerminalViewState` уже хранит `hovered_block_id`, `hovered_block_kind`, `selected_block_id` и `selected_block_kind`.
  - При начале обычного выделения текста блоковый выбор автоматически сбрасывается.
- `SnapshotView`:
  - предоставляет `blocks()`/`block_at_point()`, а `BlockSnapshot.start_line` и `SnapshotCell.point.line` находятся в одной системе координат (учитывая `display_offset`).
- `InputManager`:
  - в `handle_cursor_moved` уже обновляет `hovered_block_*` через `block_at_point`.
  - игнорирует клики по `BlockKind::Prompt`, поэтому пользователь может выбрать только `Command` или `FullScreen`.
  - реализует `BindingAction::Copy` для копирования выделенного текста.
  - имеет доступ к `SnapshotOwned` через `terminal_state` и умеет работать с clipboard.

## Что нужно сделать

1. Расширить `BindingAction`:
   - Добавить новый вариант `CopyBlock`.
   - Прописать дефолтный биндинг в `platform_keyboard_bindings()` — `Cmd+Shift+B` (macOS) / `Ctrl+Shift+B` (остальные платформы) подходит и не конфликтует с текущим `Copy`.
   - Убедиться, что новые биндинги подхватываются в `BindingsLayout::new()` и тесты `bindings.rs` (которые итерируют все биндинги) по‑прежнему проходят.

2. Обработка `CopyBlock` в `InputManager::handle_keyboard_event`:
   - При получении `BindingAction::CopyBlock`:
     - получить `SnapshotView` (`let snapshot = terminal_state.view();`).
     - проверить, что `view_state.selected_block_id` и `view_state.selected_block_kind` заданы и тип ≠ `BlockKind::Prompt`; иначе ничего не делать.
     - найти `BlockSnapshot` с нужным `id` через `snapshot.blocks()`. Если `line_count == 0`, значит блок вне текущего viewport — тоже ничего не делаем.
     - Собрать текст:
       - диапазон линий задаётся как `start_line..start_line + line_count`;
       - проитерироваться по `snapshot.cells`, отфильтровав ячейки с `point.line` внутри диапазона;
       - пропустить ячейки с `Flags::WIDE_CHAR_SPACER`, чтобы не дублировать широкие символы;
       - сгруппировать символы по строкам, обрезая хвостовые пробелы и добавляя `\n` между строками (последнюю строку можно оставить без завершающего перевода).
       - результат — обычный `String`.
     - записать строку в clipboard: `clipboard.write(ClipboardKind::Standard, text);`.
     - вернуть `iced::event::Status::Captured`, чтобы хоткей считался обработанным.

3. Поведение для различных типов блоков:
   - `Command` и `FullScreen` копируются как описано выше.
   - `Prompt` и любые будущие «не копируемые» типы (если появятся) должны приводить к no-op. Это лучше явно отразить в коде (guards + комментарий), чтобы следующий разработчик не пытался добавить копирование промптов на этом этапе.

## Ожидаемый результат

- Пользователь может:
  - выбрать блок (клик по нему → selected);
  - нажать горячую клавишу `CopyBlock`;
  - получить в буфере обмена текст именно этого блока (без соседних блоков); промпт-блоки игнорируются.
- Копирование работает для текущего viewport — если часть блока скрыта скроллом, копируется только видимая часть (ограничение итерации 1).

## Как протестировать

1. Юнит‑тесты:
   - В тестах `InputManager` создать `SnapshotOwned` с несколькими блоками, заполненными текстом (включая широкие символы и строки разной длины).
   - Установить `selected_block_id`/`selected_block_kind` на копируемый блок, вызвать обработчик `CopyBlock`, проверить содержимое тестового clipboard.
   - Отдельные тесты: отсутствие выбранного блока → clipboard не меняется; выбран `Prompt` → clipboard не меняется.

2. Ручной тест:
   - Запустить терминал, выполнить несколько команд (создать несколько блоков);
   - выбрать блок кликом;
   - нажать биндинг `CopyBlock`;
   - вставить содержимое в текстовый редактор и убедиться, что скопировался только выбранный блок.
