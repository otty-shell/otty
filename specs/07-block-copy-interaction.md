# 07. Минимальное взаимодействие: копирование активного блока

## Обзор

Седьмая задача: добавить простое действие над активным блоком — копирование содержимого выбранного блока в буфер обмена. Используется `selected_block_id` из `TerminalViewState` и данные snapshot’а.

См. спецификацию: `specs/spec.md`, раздел 8.3.

## Что уже есть

- Задачи 01–06:
  - UI умеет отображать рамки блоков и хранит `hovered_block_id` / `selected_block_id`.
- `InputManager`:
  - уже реализует действия `BindingAction::Copy` для копирования выделенного текста.
  - имеет доступ к `SnapshotOwned` через `terminal_state` и умеет работать с clipboard.

## Что нужно сделать

1. Расширить `BindingAction`:
   - Добавить новый вариант `CopyBlock`.
   - В `BindingsLayout` определить дефолтный биндинг (например, `Cmd+Shift+B` или иной не конфликтующий с существующим `Copy`).

2. Обработка `CopyBlock` в `InputManager::handle_keyboard_event`:
   - При получении `BindingAction::CopyBlock`:
     - получить `SnapshotView` (`let terminal_state_ref = terminal_state.view();`);
     - определить выбранный блок:
       - `let selected = view_state.selected_block_id.as_ref();`;
       - найти `BlockSnapshot` с таким `id` в `terminal_state_ref.blocks()`.
     - если блок найден:
       - пройти по `terminal_state_ref.cells` и собрать символы, попадающие в диапазон линий блока (`start_line..start_line + line_count`);
       - аккуратно вставлять `\n` на границе линий;
       - получить итоговую строку.
     - записать строку в clipboard:
       - `clipboard.write(ClipboardKind::Standard, text);`.

3. Поведение для различных типов блоков:
   - На этой итерации:
     - `Command` и `FullScreen` — копируются как обычный текст блока;
     - `Prompt` — можно либо копировать, либо временно игнорировать (решить и описать в коде/комментарии).

## Ожидаемый результат

- Пользователь может:
  - выбрать блок (клик по нему → selected);
  - нажать горячую клавишу `CopyBlock`;
  - получить в буфере обмена текст именно этого блока (без соседних блоков).

## Как протестировать

1. Юнит‑тесты:
   - В тестах `InputManager` создать `SnapshotOwned` с несколькими блоками и разными символами;
   - установить `selected_block_id` на один из блоков;
   - вызвать обработчик `CopyBlock` и проверить, что в тестовом clipboard оказалась ожидаемая строка.

2. Ручной тест:
   - Запустить терминал, выполнить несколько команд (создать несколько блоков);
   - выбрать блок кликом;
   - нажать биндинг `CopyBlock`;
   - вставить содержимое в текстовый редактор и убедиться, что скопировался только выбранный блок.
